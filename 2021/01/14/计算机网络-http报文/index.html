计算机网络-http报文

计算机网络-http报文

1、http报文概念

http报文就是http通信的基本单位,由8位字节流组成

2、http报文的整体结构

整体结构:报文首部 + 空行 + 报文实体
对于请求报文和响应报文又有区别

请求报文

报文首部: 请求方法 + URI + HTTP版本 + HTTP首部字段
空行: CR + LF
报文实体: ….

响应报文

报文首部: HTTP版本 + 状态码 + HTTP首部字段
空行: CR + LF
报文实体: ….

3、HTTP报文具体组成

报文首部

报文首部主要可以区分为两块 起始行 和 http首部字段

起始行

1、请求报文的起始行:请求行
方法名 + URI + HTTP版本
例如:GET /index HTTP1.1

2、响应报文的起始行:状态行
HTTP版本 + 状态码 + 原因短语
例如 HTTP1.1 200 OK

http首部字段

这一部分在下文详细列举

空行

CR 或 LF (回车或换行符)
在报文中,空行是头部和实体的分界线,主要作用就是用于区分头部和实体,如果在头部字段中故意额外添加一个空行的话,那么接下来所有的内容全部都会被当作实体来看待

报文实体

请求体/响应体:一般请求报文的实体部分不放东西,而响应实体则会放一些响应的结果

4、首部字段

4.1 首部字段组成方式

  • http首部字段的组成通常是key:value的形式的键值对
    如:Content-type:text/html
  • 但是也有可能一个key值会有多这个value值-> key:value1,value2
    多个选项的value之间会使用”,”隔开
    如:Keep-Alive:timeout=15,max=100
  • 还有可能一个key会有可选参数 -> key- value1,q=1;value2,q=0.8
    如:Accept:text/html,q=1;application/xml,q=0.8 (这里有两个可选值,这里的q的意思是权重,优先使用哪个,范围是0-1,值越大权重越大)

4.2 首部字段的类型

基本可以分为4类

  • 1.通用首部字段:请求报文和响应报文中都会有的字段类型
  • 2.请求首部字段:请求报文的字段类型
  • 3.响应首部字段:响应报文的字段类型
  • 4.实体首部字段:报文实体部分的字段类型

这里需要记住每种类型中常用的字段

1.通用首部字段

  • 1.Cache-Control

  • 2.Date
    用于表示创建报文的时间和日期
    例:Date:Sun,05 Jar 2021 19:00:00 GMT

  • 3.Connection
    作用有两个:
    1、控制不再转发代理的首部字段:Connection: …(一些不会转发代理的字段名称)
    2、管理持久连接: Connection:Keep-Alive (在http/1.1 之前默认是close,在http/1.1之后默认就是Keep-Alive)

    这里还有一个首部字段的key为Keep-Alive
    可以用来设置最大的请求数和超时时间
    Keep-Alive:timeout=5,max=1000 (timeout代表一个空闲连接需要保持打开状态的最小时长——单位秒,max代表在此连接可以发送请求的最大值)
    这个字段设置的值的前提是需要设置Connection:Keep-Alive,不然设置的没有意义

  • 4.Transfer-Encoding
    作用: 规定传输的报文中主体部分使用的编码方式
    例: Transfer-Encoding:chunked(代表分块出传输)
